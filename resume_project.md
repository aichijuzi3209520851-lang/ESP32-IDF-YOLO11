# 项目经验 — 基于ESP32-S3的太阳能板缺陷检测边缘AI系统

## 项目概述

独立设计并实现了一套**端到端边缘AI缺陷检测系统**，在ESP32-S3微控制器上完成YOLO11n模型的训练、量化、部署与推理全流程，实现太阳能板6类缺陷的实时检测与Web可视化监控。

## 核心技术栈

**MCU**: ESP32-S3 (240MHz双核, 8MB PSRAM, 16MB Flash) |
**模型**: YOLO11n (2.6M参数, INT8量化) |
**量化**: ESP-PPQ (Power-of-2 power-of-two 量化) |
**推理框架**: ESP-DL v3.3 (FlatBuffers模型格式, 双核调度, SIMD加速) |
**前端**: Chart.js + MJPEG 实时流仪表盘 | **语言**: C/C++ (固件), Python (训练管线)

## 关键成果

- **模型精度**: mAP@0.5 **0.577**, Precision **0.715**, Recall **0.576** (6类缺陷, 2025张训练数据)
- **推理速度**: ESP32-S3双核 **~4.6秒/帧** (320×320, INT8量化)
- **模型压缩**: 浮点模型 5.2MB → INT8量化 **2.8MB** (54%), 零精度损失验证 (cosine相似度 0.9899)
- **视频流**: MJPEG 实时流, 零拷贝双缓冲PSRAM架构, 事件驱动信号量唤醒

## 主要工作

### 1. 模型训练与量化全流程
- 使用Ultralytics YOLO11n在2025张太阳能板图像上训练6类缺陷检测模型
- 自定义ESP-DL兼容的Detect头输出结构 (box/score分离, DFL后处理外移)
- 通过ESP-PPQ完成INT8 Power-of-2量化, 输出.espdl标准模型格式
- 编写自动化训练→导出→量化→部署的一键脚本 (`train_and_export.py`)

### 2. 边缘推理引擎实现
- 从零实现YOLO11检测器的C++推理管线: JPEG解码 → Letterbox预处理 → HWC→CHW转置 → INT8量化 → 模型推理 → DFL分布解码 → NMS后处理
- 深入理解并解决INT8量化域的**反量化**问题: 输入归一化、输出分数和边界框的指数缩放
- 调试并修复HWC交错与CHW平面格式的通道顺序错误, 使模型输出从全零恢复为正确检测
- 实现`model->test()`板端推理正确性验证 (嵌入测试真值的端到端一致性校验)

### 3. 视频流与Web仪表盘
- 设计双缓冲+信号量的**零拷贝MJPEG流架构**, 消除重复拷贝, 降低PSRAM争用
- 实现事件驱动流推送: 主循环写缓冲区→信号量唤醒HTTP任务→直接从PSRAM发送
- 构建单文件Web仪表盘: Chart.js实时趋势图 + MJPEG视频流 + AI推理结果自动刷新

### 4. 嵌入式系统深度调优
- **PSRAM内存管理**: 通过预分配RGB888缓冲区、param_copy策略选择解决8MB PSRAM的碎片化问题
- **模型加载策略**: 对比FLASH_RODATA与param_copy模式的性能与内存权衡
- **帧缓冲溢出处理**: 分析DMA帧率与消费速率不匹配, 采用`CAMERA_GRAB_WHEN_EMPTY`从根源消除FB-OVF
- **I2C总线管理**: 集成SSD1306 OLED (chill-sam/ssd1306库)、SHT20温湿度、TSL2584光照等多设备

### 5. 推理精度调试 (完整故障排查链)
- 量化一致性验证: FP32 ONNX vs INT8 ESP-DL 输出cosine相似度 0.9899
- 逐层误差分析: 识别score_scale=8导致的INT8置信度分辨率粗化问题
- 建立`model->test()`板端测试、反量化校验、输入归一化验证的分层调试策略

## 项目亮点

1. **全栈能力**: 从PyTorch训练→ONNX导出→ESP-PPQ量化→C++推理引擎→Web前端, 覆盖完整的边缘AI部署链
2. **深度调优经验**: 解决了PSRAM碎片化、HWC/CHW通道错序、INT8量化域反量化、DMA帧缓冲溢出等多个底层嵌入式问题
3. **工程化**: 双缓冲零拷贝流、FreeRTOS多任务协作(采集/推理/HTTP/OLED)、一键训练部署脚本
4. **端到端验证**: 建立从模型精度测试→板端推理校验→Web可视化的完整质量保障链路
