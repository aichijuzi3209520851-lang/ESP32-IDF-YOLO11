# ESP32-S3 太阳能板智能检测相机 产品说明书

## 1. 产品概述

ESP32-S3 太阳能板智能检测相机是一款面向光伏运维的边缘智能监测终端。产品基于 ESP32-S3 MCU 与 OV5640 摄像头，融合本地 YOLO11n 检测模型、MJPEG 实时视频流、环境传感与 OLED 本地显示，实现太阳能板表面缺陷的自动识别与可视化展示。

## 2. 目标用户与应用场景

- 光伏电站运维单位
- 分布式屋顶太阳能场站
- 巡检无人机/无人车边缘检测节点
- 太阳能电站故障预警与质量检测
- 现场运维监控与远程巡检

## 3. 核心功能

### 3.1 实时缺陷检测

- 支持 6 类太阳能板缺陷识别：
  - 鸟粪
  - 灰尘
  - 积雪
  - 电损伤
  - 物理损伤
  - 清洁
- 使用轻量级 YOLO11n INT8 模型，适配 ESP32-S3 本地推理
- 具备检测结果绘制与标注输出

### 3.2 视频流与 Web 可视化

- 提供原始 MJPEG 视频流
- 支持带检测标注的图像输出
- 内置 Web 仪表盘，浏览器访问即可查看实时画面与结果
- 端点说明：
  - `/`：设备仪表盘
  - `/stream`：MJPEG 实时视频流
  - `/annotated`：检测标注图像

### 3.3 环境监测

- 温湿度监测：SHT20
- 光照强度监测：TSL2584
- 系统电压监测：MAX471
- 结合图像检测，实现多维健康状态判断

### 3.4 本地显示与状态指示

- SSD1306 OLED 显示运行状态、检测结果与环境数据
- LED 指示设备工作状态
- 本地显示支持即时反馈，适用于现场设备查看

## 4. 产品优势

- 边缘 AI：本地离线推理，无需依赖云端
- 低成本部署：基于 ESP32-S3 的嵌入式方案
- 真实场景适配：适用于光伏板巡检、运维监测与异常识别
- 系统集成化：图像采集、模型推理、传感器检测、网络展示一体化
- 可扩展性强：支持模型迭代、传感器扩展与功能定制

## 5. 硬件规格

- MCU：ESP32-S3-WROOM-1
- 主频：240MHz 双核
- 内存：8MB PSRAM + 16MB Flash
- 摄像头：OV5640，320×320 JPEG
- OLED：SSD1306 128×64 I2C
- 温湿度传感器：SHT20
- 光照传感器：TSL2584
- 电压检测：MAX471 via ADC1_CH0
- 模型：YOLO11n INT8（ESP-DL 格式）
- 推理耗时：约 4.6 秒 / 帧

## 6. 系统架构与模块说明

### 6.1 图像采集与流式传输

- OV5640 摄像头采集 JPEG 并存储到 PSRAM 双缓冲区
- 通过 HTTP 服务输出 MJPEG 实时视频流
- 支持双路画面展示：原始图像与检测结果

### 6.2 本地推理流程

- JPEG 解码 -> RGB888 格式转换
- Letterbox 图像尺寸调整
- HWC->CHW 数据排列
- INT8 量化输入
- ESP-DL 模型推理
- DFL 解码 + NMS 后处理

### 6.3 传感与显示

- I2C 总线读取 SHT20、TSL2584、SSD1306
- ADC 读取 MAX471 电压值
- OLED 显示设备状态与检测摘要

## 7. 接口与引脚分配

### 7.1 摄像头接口

- XCLK：GPIO15
- SCCB_SDA：GPIO4
- SCCB_SCL：GPIO5
- D0：GPIO11
- D1：GPIO9
- D2：GPIO8
- D3：GPIO10
- D4：GPIO12
- D5：GPIO18
- D6：GPIO17
- D7：GPIO16
- VSYNC：GPIO6
- HREF：GPIO7
- PCLK：GPIO13

### 7.2 I2C 总线

- SDA：GPIO2
- SCL：GPIO3
- 设备地址：
  - SHT20：0x40
  - TSL2584：0x39
  - SSD1306：0x3C / 0x3D

### 7.3 ADC 电压检测

- ADC1_CH0：GPIO1

### 7.4 LED 指示

- 默认 GPIO8，可通过 `idf.py menuconfig` 进行配置

## 8. 使用说明

### 8.1 系统部署

1. 使用 ESP-IDF 环境编译固件
2. 将固件烧录到 ESP32-S3 设备
3. 设备连接光伏板检测现场电源与摄像头
4. 启动后通过浏览器访问设备 IP

### 8.2 访问方式

- Web 仪表盘：`http://<设备IP>/`
- 实时视频：`http://<设备IP>/stream`
- 检测标注图：`http://<设备IP>/annotated`

### 8.3 调试与维护

- 串口调试：`idf.py -p COM<X> flash monitor`
- 退出监控：`Ctrl+]`
- 快速清理：`idf.py fullclean`
- 重新编译：`idf.py build`

## 9. 模型与训练

- 模型架构：YOLO11n
- 输入尺寸：320×320
- 量化方式：INT8（ESP-PPQ）
- 模型格式：ESP-DL `.espdl`
- 训练流程：`
  python train_and_export.py`
- 自动流程包含：训练、ONNX 导出、量化、模型部署
