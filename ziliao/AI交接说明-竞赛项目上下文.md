## 2. 当前项目真实情况

这是一个 ESP-IDF 工程，目标板为 ESP32-S3。

项目当前真实实现更接近：

```text
ESP32-S3 + OV5640 摄像头 + YOLO11n ESP-DL 本地推理 + Web 视频流 + OLED/传感器接口
```

核心能力：

- ESP32-S3 本地运行 YOLO11n INT8 模型。
- 摄像头采集 320x320 JPEG 图像。
- HTTP `/stream` 输出 MJPEG 实时视频流。
- HTTP `:8080/annotated` 输出检测标注图。
- 内置 `dashboard.html` 作为 Web 仪表盘。
- SHT20、TSL2584、MAX471 驱动文件存在。
- SSD1306 OLED 驱动存在。
- LED 状态指示模块存在。
- 模型文件 `main/model/yolo11n_esp32s3.espdl` 已嵌入固件。

关键源码：

```text

main/cam/camera.c                         OV5640 摄像头初始化与采集
```

## 8. “脏污覆盖程度 + 功率衰减”决策方案

这是用户特别关心的问题。

建议算法：

### 8.1 视觉脏污指数

根据 YOLO 检测框面积占比和类别权重计算：

```text
A_i = 第 i 个检测框面积
A_total = 图像总面积
w_i = 类别权重
D_img = sum(A_i * w_i) / A_total * 100
```

类别权重建议：

```text
clean: 0.0
dust: 1.0
bird_drop: 1.5
snow_covered: 1.3
physical_damage: 2.0
electrical_damage: 2.0
```

注意：physical_damage 和 electrical_damage 应偏向告警，不应自动清洁。

### 8.2 功率衰减率

```text
P_now = 当前电压 * 当前电流
P_ref = 清洁状态或标定状态下的参考功率
P_loss = max(0, (P_ref - P_now) / P_ref * 100)
```

如果实际电流测量不稳定，视频演示中可以通过遮挡小太阳能板或调节光照模拟输出下降。

### 8.3 综合清洁评分

```text
S = 0.6 * D_img + 0.4 * P_loss
```

决策规则：

```text
S < 30: 无需清洁
30 <= S < 60: 轻度污染，启动滚刷电机
S >= 60: 重度污染，启动水泵 + 滚刷电机
温度过低: 禁止喷淋，只告警或延后清洁
光照过强: 延后湿洗，可只滚刷或等待
物理损伤/电损伤: 故障告警，不自动清洁
```

### 8.4 视频中体现方式

建议录制 4 个场景：

1. 干净状态。
   - 视觉无明显脏污。
   - 功率正常。
   - Web 显示无需清洁。

2. 只有视觉脏污。
   - 摄像头对准灰尘/鸟粪/遮挡测试图。
   - 功率衰减不明显。
   - Web 显示视觉异常、持续观察或轻度清洁。

3. 只有功率下降。
   - 遮挡小太阳能板或调暗光源。
   - 视觉检测未发现明显脏污。
   - Web 显示疑似光照变化或遮挡，暂不清洁。

4. 视觉脏污 + 功率下降。
   - 摄像头检测到脏污。
   - 功率下降。
   - 综合评分超过阈值。
   - 继电器动作，电机和水泵启动。

这样能说明系统不是单一视觉触发，也不是单一功率触发，而是多源融合决策。

## 9. 推荐 MQTT 主题设计

如果实现 MQTT，推荐如下主题：

```text
pv/device/{device_id}/telemetry     # 温湿度、光照、电压、电流、功率
pv/device/{device_id}/vision        # 检测类别、置信度、检测框、视觉脏污指数
pv/device/{device_id}/decision      # 综合评分、清洁建议、禁止清洁原因
pv/device/{device_id}/state         # 设备状态、执行器状态、联网状态
pv/device/{device_id}/cmd           # 远程控制命令
```
